2.1.3 Logging

When it encounters a difficulty, Polipo will print a friendly message. The location where these messages go is controlled by the configuration variable logFile. If it is empty, messages go to the error output of the process (normally the terminal); otherwise, it is the name of a file where all output will accumulate.

The variable logFile defaults to empty if daemonise is false, and to /var/log/polipo otherwise.

The amount of logging is controlled by the variable logLevel. Please see the file log.h in the Polipo sources for the possible values of logLevel.
